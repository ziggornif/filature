## Agent Brief — 03a Spools (core: model, add, edit, list, view)

**Category:** feature
**Summary:** Introduce the `Spool` domain and its create/read surface — add a spool, edit its descriptive fields, list spools (filter + sort, no reload), and view a spool's detail with derived Remaining Weight / Ratio / Length.

**Slice / context:**
First increment of the `spools` slice — the domain-heavy slice of Filature. The
`materials` referential already exists (`MaterialRepository`, seeded at startup)
and is the source of truth for density. This increment builds the Spool
aggregate and its create/read use cases. Spool *operations* — adjust-weight
(set-remaining / consume), archive, Stock Value, status transitions
Open/Empty/Archived — are the **next** increment (03b) and are out of scope here.

A Spool is made of one Material, has a real Colour, a Diameter, a Net Weight, a
Remaining Weight, a Price Paid, and a Spool Status (see `docs/glossary.md`).
Weight modelling follows **ADR-0004**: Net Weight is entered directly from the
label; there is no Tare and no weigh-total flow.

**Desired behavior:**

- **Add a spool.** Operator supplies: Material (chosen from the referential),
  Colour (hex `#RRGGBB` + optional name), Diameter (1.75 or 2.85 mm), Net Weight
  (grams, > 0), Price Paid (money, ≥ 0). On creation: `remaining_weight = net_weight`,
  Spool Status = **Sealed**, id and creation timestamp are generated by the
  persistence adapter (ULID id, opaque to the domain — same pattern as
  `MaterialId`). Adding a spool whose `material_id` is not in the referential is
  rejected (referential integrity), surfaced as a domain/validation error, not a
  raw DB error.
- **Edit a spool.** Operator may change the descriptive fields: Material, Colour,
  Diameter, Net Weight, Price Paid. **Remaining Weight and Spool Status are not
  editable here** (they are operational, owned by 03b). Edge case: if an edit
  lowers Net Weight below the current Remaining Weight, Remaining Weight is
  **clamped** down to the new Net Weight.
- **List spools.** Returns all non-archived spools (Archived exists as a status
  but nothing archives in 03a, so effectively all). Supports **filter** (at least
  by Material, and by Spool Status) and **sort** (at least by creation time and by
  Remaining Ratio), applied without a full-page reload (htmx). Each row shows the
  Colour swatch, Material name, Diameter, Remaining Weight, Remaining Ratio, and
  Remaining Length.
- **View a spool.** Detail page for one spool showing every stored field plus the
  derived Remaining Ratio and Remaining Length, and the Colour swatch.
- **Derived quantities (pure, never stored):**
  - **Remaining Ratio** = Remaining Weight ÷ Net Weight (reuse `Grams::ratio_of`).
  - **Remaining Length** (metres) = `mass / (density × π · (d/2)²)`, from the
    Material's density (g/cm³) and the Spool's Diameter. Pure function; guard the
    density divide (density is now guaranteed finite and > 0 — see the TD-004
    prerequisite below).
- **Invariants / edge cases:**
  - `Colour` rejects a malformed hex (must be `#` + 6 hex digits); name optional.
  - `Diameter` is a closed set {1.75, 2.85}; no other value representable.
  - `Net Weight` must be > 0; `Remaining Weight` in `0..=net`; `Price Paid` ≥ 0.
  - i18n: all UI strings via the existing en/fr catalogs — no hardcoded strings
    (ADR-0001). New keys added to both locales.

**Prerequisite folded into this brief (TD-004):**
Before Remaining Length divides by density, tighten the `materials` domain:
`Density::new` must reject non-finite values (`v.is_finite() && v > 0.0`), and a
validated **MaterialName** must reject blank/whitespace-only names. Update
`docs/tech-debt.md` to move TD-004 to Resolved when done.

**Key interfaces:** (glossary + API/SPI terms — no file paths)
- `Spool` — new aggregate: `id: SpoolId`, `material_id: MaterialId`, `colour: Colour`,
  `diameter: Diameter`, `net_weight: Grams`, `remaining_weight: Grams`,
  `price_paid: Money`, `status: SpoolStatus`, plus a creation timestamp.
- `NewSpool` — creation input: `material_id`, `colour`, `diameter`, `net_weight`,
  `price_paid` (remaining/status/id/timestamp derived, not supplied).
- `Colour` — value object: validated hex + optional name.
- `Diameter` — enum {`Mm1_75`, `Mm2_85`}.
- `SpoolStatus` — enum {`Sealed`, `Open`, `Empty`, `Archived`}; only `Sealed` is
  produced in 03a.
- `spools` **API port** — `add(NewSpool)`, `edit(Spool)`, `list(filter, sort)`,
  `view(SpoolId)`. Reuse the existing shared value objects (`Grams`, `Money`) from
  `shared/`.
- `SpoolRepository` **(SPI)** — `insert`, `update`, `list`, `get`. Cross-slice
  rule (see `docs/architecture.md`): list/view return a **read model** joined by
  the adapter that carries the Material's `name` + `density` (for Remaining Length
  / display) — the `spools` slice must **not** `use` the `materials` slice; the
  shared cross-slice type lives in `shared/`. Id + timestamp are generated in the
  adapter (materials precedent), so no separate `Clock`/`IdGenerator` port unless
  a use case needs injectable time.

**Acceptance criteria:**
- [ ] `Density::new` rejects non-finite input; a validated `MaterialName` rejects
      blank/whitespace-only. TD-004 moved to Resolved in `docs/tech-debt.md`.
- [ ] `Colour` rejects non-`#RRGGBB` hex; accepts an optional name.
- [ ] `Diameter` cannot hold a value outside {1.75, 2.85}.
- [ ] Adding a spool persists it with `remaining = net`, status `Sealed`, an
      adapter-generated ULID id and timestamp.
- [ ] Adding a spool with an unknown `material_id` is rejected (not a raw DB error).
- [ ] Editing descriptive fields updates them; editing net below remaining clamps
      remaining to the new net; edit leaves remaining/status otherwise untouched.
- [ ] `list` returns spools filtered by Material and by Status and sorted by
      creation time and by Remaining Ratio; the table updates via htmx without a
      full reload.
- [ ] Remaining Ratio = remaining ÷ net; Remaining Length matches
      `mass/(density·π·(d/2)²)` for both diameters (unit-tested with known values).
- [ ] `view` renders all fields + Colour swatch + derived Ratio & Length.
- [ ] All new UI strings exist in both en and fr; none hardcoded.
- [ ] Tests at every layer: domain (invariants, derived, clamp), use cases via SPI
      stubs, SPI adapter integration (Testcontainers Postgres), e2e (add → list →
      view). Offline `.sqlx` metadata regenerated so CI builds with `SQLX_OFFLINE`.

**Out of scope:**
- adjust-weight (set-remaining / consume), archive, Stock Value, and status
  transitions to Open/Empty/Archived — these are **03b**.
- Any **Location** reference on a spool — the `locations` slice does not exist yet;
  no location FK/field is added now (added when that slice lands).
- Delete of a spool (retirement is Archive, in 03b).
- Dashboard aggregates (separate slice).
- Any tare / weigh-total input (dropped, ADR-0004).
